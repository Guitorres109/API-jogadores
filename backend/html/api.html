Documentação da API de Jogadores

API para consultar informações de jogadores e imagens aleatórias

Base URL

Supondo que a aplicação esteja rodando localmente:

http://SEU_IP:3000

Endpoints

1. Verificar status do servidor

GET /verificar

Verifica se o servidor está online.

Resposta de sucesso:

{
    "status": "success",
    "message": "Servidor está online e respondendo"
}

2. Jogador aleatório

GET /api/jogadores/aleatorio

Retorna um jogador aleatório com uma foto aleatória.

Exemplo de resposta:

{
    "status": "success",
    "message": "Jogador: neymar\nIdade: 31\nTime: PSG\nNacionalidade: Brasil",
    "foto": "http://SEU_IP:3000/img/neymar1.jpg"
}

3. Pesquisar jogador por nome

GET /api/jogadores/:nome

Substitua :nome pelo nome do jogador (em letras minúsculas).

Exemplo: /api/jogadores/neymar

Resposta de sucesso:

{
    "status": "success",
    "message": "Jogador: neymar\nIdade: 31\nTime: PSG\nNacionalidade: Brasil",
    "foto": "http://SEU_IP:3000/img/neymar1.jpg"
}

Resposta se o jogador não for encontrado:

{
    "status": "error",
    "message": "Jogador \"nome\" não encontrado"
}

Servindo imagens

As imagens dos jogadores ficam disponíveis em:

GET /img/NOME_DA_IMAGEM.jpg

Observações

  • O IP do servidor deve ser obtido dinamicamente (pode ser o localhost ou IP da rede local).
  • Os nomes dos jogadores no JSON devem estar em letras minúsculas para a pesquisa funcionar corretamente.
  • O endpoint aleatório retorna também uma imagem aleatória do jogador.
